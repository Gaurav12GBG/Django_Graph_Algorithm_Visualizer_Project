Explore more about the Graph Algorithm Visualizer

Inspiration

Graph algorithms can be challenging to understand for the average CS student. visualizing it on the whiteboard often hurts more than it helps, and the specific example videos on youtube sometimes aren't enough. The goal of this project will to create an all-in-one website that a student as well as teachers can access to reference algorithms they are confused about.

What it does

The web app we created allows the user to create a graph of their choosing. Once they have done so, they can toggle over to the algorithms tab to simulate an algorithmon their graph, or gain information about their graph. Currently the following are available to the users:
# An iterative simulation of DFS, BFS.
# A list of the cut vertices of the graph.
# A minimum spanning tree of the graph(found with krushkal's algorithm)

How we built it

Basically, we used pure Vanilla Javascript, Python Django Framework to set the functional or backend environment, which reduces the lot of the clutter that bigger projects normally come with. The UI/UX designed are done through the Bootstrap Framework. The graph itself simulated through a visualization libraries called cytoscape.

Challenges we ran into

We tried to code things out as quickly as possible. This turned out to be an issue, however, since our code quickly become cluttered and difficult to read. States were beginning to interweave between components, and connecting UI to actual graph fucntions was proving to be a difficult task. It was also difficult to figuring out how to extract the data from the cytoscape graph, since we were not used to the library.

What we learned

In this project, we learned how to create a semi-stable, professional-ish Django project than can somewhat maintain itself. Mostly the javascript and django libraries took care of many trivial errors we would have to deal with, and the unified code style made it really easy for all of us to understand each other's code. We also gained a deeper understanding of how django featues/libraries works through this.

What's next for Graph Algorithm Visualizer

Adding more algorithms in an iterative version is the next logical step for us, since we wanted to focus on showing how graph algorithms operate. Aside from that, exploring better UX for creating graphs would be a really nice improvement.
